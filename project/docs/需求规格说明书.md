# 需求规格说明书（SRS）

项目：酒店运营支持系统（SpringBoot + Vue + RAG）

## 1. 业务目标

- 提升员工查询规范与应急流程的效率。
- 提升管理层对运营指标异常的识别和响应能力。
- 建立可持续维护的酒店知识资产。

## 2. 用户角色

- 员工（STAFF）：提问、查看规范建议。
- 管理员（ADMIN）：管理知识库与系统配置。
- 管理层（MANAGER，可扩展）：查看看板与决策建议。

## 3. 功能需求

- F1 登录鉴权：用户登录并获取访问 JWT。
- F2 智能问答：输入业务问题，返回答案与引用来源。
- F3 知识管理：新增、查看、检索知识文档。
- F4 运营看板：展示入住率、营收、评分、差评率与趋势。
- F5 异常提示：输出当前运营建议文本。

## 4. 非功能需求

- N1 可用性：核心接口可稳定访问。
- N2 安全性：受保护接口必须校验 JWT。
- N3 可维护性：模块边界清晰，接口文档完整。
- N4 可扩展性：可替换为向量检索与真实 LLM 推理。

## 5. 验收标准

- A1 后端编译通过、测试通过。
- A2 前端构建通过，可访问页面并完成联调。
- A3 登录后可调用问答、知识、看板接口。
- A4 文档齐全：接口、部署、测试、设计说明齐备。
